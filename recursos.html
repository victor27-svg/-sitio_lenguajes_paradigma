Material complementario para profundizar: glosario, buenas prácticas, retos y enlaces.

Glosario rápido

Inmutabilidad
Propiedad de no cambiar el estado una vez creado; clave en programación funcional.
Abstracción
Proceso de ocultar detalles internos y exponer una interfaz más simple.
Acoplamiento
Grado de dependencia entre módulos. Buscamos bajo acoplamiento.
Cohesión
Grado de relación lógica dentro de un módulo. Buscamos alta cohesión.
Concurrencia
Ejecución de múltiples tareas en períodos superpuestos de tiempo.

Buenas prácticas

  • Aplica principios SOLID en OO.
  • Prefiere funciones puras y pruebas unitarias.
  • Diseña APIs con contratos claros (REST/GraphQL).
  • Automatiza CI/CD y añade linting/formateo.
  • Protege datos y secretos (12-factor, .env).

Retos sugeridos

  1. Implementa map, filter, reduce desde cero.
  2. Crea una API REST con autenticación y pruebas.
  3. Resuelve un problema con enfoques distintos (imperativo vs funcional).
  4. Programa un productor/consumidor concurrente.
